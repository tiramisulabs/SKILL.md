# Components v2

Original source URL: https://seyfert-web-git-seyfert-v5-tiramisulabs.vercel.app/docs/learn/components/v2
Coverage reference: components.md (+ builders.md)
Verification status: Source-verified (seyfert-core, branch more-qol)

## Page Summary

Discord Components v2 replace the old content/embed/component layout rules with fully composable layout builders. Seyfert ships dedicated builders (`Container`, `Section`, `TextDisplay`, `Thumbnail`, `MediaGallery` + `MediaGalleryItem`, `Separator`, `File`) plus the `Spacing` enum, all exported from the root `'seyfert'`. To render v2 components you MUST set the message flag `MessageFlags.IsComponentsV2` (`1 << 15`); Seyfert does NOT auto-apply it. Discord additionally rejects a v2 message that also carries `content`, `embeds`, `stickers` or `poll` â€” once `IsComponentsV2` is set, every piece of visible text must live inside a `TextDisplay`.

## Key APIs (verified)

All builders extend `BaseComponentBuilder` (`src/builders/Base.ts`, whose base `toJSON()` just spreads `this.data`) and are exported from `'seyfert'` (`src/builders/index.ts`, re-exported by `src/index.ts`).

- `Container` (src/builders/Container.ts)
  - `addComponents(...RestOrArray<ContainerBuilderComponents>)`, `setComponents(...)` â€” children type `ActionRow | TextDisplay | Section | MediaGallery | Separator | File`.
  - `setColor(color: ColorResolvable)` -> sets `accent_color` (accepts hex string, color name, or int via `resolveColor`; `resolveColor` THROWS on a malformed `#hex` in v5).
  - `setSpoiler(spoiler = true)`, `setId(id: number)`, `toJSON()` (maps each child's `toJSON()`).
- `Section` (src/builders/Section.ts) â€” generic `Section<Ac extends Button | Thumbnail = Button | Thumbnail>`.
  - `addComponents(...RestOrArray<TextDisplay>)`, `setComponents(...)` â€” children are `TextDisplay` only (1-3 per Discord).
  - `setAccessory(accessory: Ac)` â€” exactly one `Button` OR `Thumbnail`.
  - `toJSON()` THROWS `SeyfertError('MISSING_ACCESSORY')` if no accessory was set.
- `TextDisplay` (src/builders/TextDisplay.ts) â€” `setContent(content: string)`, `setId(id)`. (No `setMedia`. Markdown works inside `content`.)
- `Thumbnail` (src/builders/Thumbnail.ts) â€” `setMedia(url: string)` -> `data.media = { url }`, `setDescription(description: string | undefined)` (alt text), `setSpoiler(spoiler = true)`, `setId(id)`.
- `MediaGallery` (src/builders/MediaGallery.ts) â€” `addItems(...RestOrArray<MediaGalleryItem>)`, `setItems(...)`, `setId(id)`, `toJSON()`. Discord allows 1-10 items (not enforced client-side).
- `MediaGalleryItem` (src/builders/MediaGallery.ts) â€” plain class (NOT a `BaseComponentBuilder`, no `setId`): `setMedia(url)` -> `data.media = { url }`, `setDescription(desc)`, `setSpoiler(spoiler = true)`, `toJSON()` THROWS `SeyfertError('MISSING_MEDIA')` if media unset.
- `Separator` (src/builders/Separator.ts) â€” `setSpacing(spacing: Spacing)`, `setDivider(divider = false)`, `setId(id)`. (No `toJSON` override â€” base spread is fine; nothing to validate.)
- `File` (src/builders/File.ts) â€” `setMedia(url: string)` -> `data.file = { url }` (use `attachment://name` or a plain name; references an uploaded attachment), `setSpoiler(spoiler = true)`, `setId(id)`.
- `Spacing` enum (src/types/payloads/components.ts:570) â€” ONLY `Small = 1`, `Large = 2`. There is NO `Spacing.None` / `Spacing.Medium`.
- `MessageFlags.IsComponentsV2 = 1 << 15` (src/types/payloads/channel.ts:835).
- `ComponentType` (src/types/payloads/components.ts:18) â€” `Section = 9`, `TextDisplay = 10`, `Thumbnail = 11`, `MediaGallery = 12`, `File = 13`, `Separator = 14`, `Container`, `FileUpload`. Builders set their own `type` in the constructor.
- `FileUpload` (src/builders/FileUpload.ts) â€” NEW v2 builder, but it is a MODAL input component (`setCustomId`, `setMinValues`, `setMaxValues`, `setRequired`, `setId`), NOT a message-layout component; not used in the message `components` array.

## Code Examples (verified)

### 1. Container assembling every child type

```ts
import {
  Container, Section, Thumbnail, TextDisplay,
  File, Separator, MediaGallery, MediaGalleryItem,
  Spacing, MessageFlags,
} from 'seyfert';
// ctx: CommandContext

const section = new Section()
  .setAccessory(
    new Thumbnail()
      .setMedia('https://example.com/image.png')
      .setDescription('This is a thumbnail image'),
  )
  .addComponents(new TextDisplay().setContent('A message with a thumbnail'));

const container = new Container()
  .addComponents(
    section,
    new File().setMedia('attachment://example.png').setSpoiler(false),
    new Separator().setSpacing(Spacing.Small).setDivider(true),
    new MediaGallery().addItems(
      new MediaGalleryItem().setMedia('https://example.com/1.png').setDescription('Image 1'),
      new MediaGalleryItem().setMedia('https://example.com/2.jpg').setSpoiler(),
    ),
  )
  .setColor('#5865f2')
  .setSpoiler(false);

// IsComponentsV2 flag is REQUIRED for v2 components to render.
await ctx.write({ components: [container], flags: MessageFlags.IsComponentsV2 });
```

### 2. File attachment (component references an uploaded file by name)

```ts
import { File, AttachmentBuilder, MessageFlags } from 'seyfert';

const attachment = new AttachmentBuilder()
  .setName('example.png')
  .setFile('url', 'https://example.com/example.png');

const file = new File().setMedia('attachment://example.png').setSpoiler(false);

await ctx.write({
  flags: MessageFlags.IsComponentsV2, // 1 << 15
  files: [attachment],
  components: [file],
});
```

### 3. Section with a Button accessory + interactive buttons via ActionRow

A `Section` accessory is exactly one `Button` or `Thumbnail`. To place a *row* of buttons, wrap them in an `ActionRow` (a valid `Container` child). `Container` does NOT accept a bare `Button`.

```ts
import {
  Container, Section, TextDisplay, ActionRow, Button,
  Separator, ButtonStyle, MessageFlags,
} from 'seyfert';

const container = new Container()
  .setColor('Blurple')
  .addComponents(
    // Text + a single link button as the section accessory.
    new Section()
      .addComponents(new TextDisplay().setContent('## Docs\nRead the guide for details.'))
      .setAccessory(
        new Button()
          .setStyle(ButtonStyle.Link)
          .setLabel('Open docs')
          .setURL('https://seyfert.dev'),
      ),
    new Separator().setDivider(true),
    // A row of interactive buttons lives in an ActionRow, not the section.
    new ActionRow<Button>().addComponents(
      new Button().setCustomId('confirm').setStyle(ButtonStyle.Success).setLabel('Confirm'),
      new Button().setCustomId('cancel').setStyle(ButtonStyle.Danger).setLabel('Cancel'),
    ),
  );

await ctx.write({ components: [container], flags: MessageFlags.IsComponentsV2 });
```

### 4. Handling the interactive buttons (ComponentCommand)

The handler is identical to plain components â€” v2 layout has no effect on `customId` routing. When you `update`, keep the flag so the message stays v2.

```ts
import {
  ComponentCommand, type ComponentContext,
  Container, TextDisplay, MessageFlags,
} from 'seyfert';

export default class Confirm extends ComponentCommand {
  componentType = 'Button' as const;

  filter(ctx: ComponentContext<'Button'>) {
    return ctx.customId === 'confirm';
  }

  async run(ctx: ComponentContext<'Button'>) {
    const done = new Container()
      .setColor('Green')
      .addComponents(new TextDisplay().setContent('Confirmed.'));
    // ComponentContext.update() â€” flag must be re-sent on every v2 edit.
    await ctx.update({ components: [done], flags: MessageFlags.IsComponentsV2 });
  }
}
```

### 5. Editing a v2 message later (flag must persist)

```ts
import { Container, TextDisplay, MessageFlags } from 'seyfert';

const msg = await ctx.write(
  { components: [new Container().addComponents(new TextDisplay().setContent('Loading...'))],
    flags: MessageFlags.IsComponentsV2 },
  true, // CommandContext.write(body, withResponse) -> returns the Message
);

await ctx.editOrReply({
  components: [new Container().addComponents(new TextDisplay().setContent('Done!'))],
  flags: MessageFlags.IsComponentsV2, // omit it and Discord rejects the edit
});
```

### 6. Recreating Discord's "new components" announcement

Mirrors the upstream canonical example: banner gallery, a long markdown `TextDisplay`, link-button sections, a divider, and an attached file â€” all inside one `Container`.

```ts
import {
  Container, MediaGallery, MediaGalleryItem, TextDisplay,
  Section, Button, Separator, File, ButtonStyle, MessageFlags, AttachmentBuilder,
} from 'seyfert';

const banner = new AttachmentBuilder().setName('banner.png').setFile('url', 'https://example.com/banner.png');

const container = new Container().addComponents(
  new MediaGallery().addItems(new MediaGalleryItem().setMedia('attachment://banner.png')),
  new TextDisplay().setContent(
    '## Introducing New Components for Messages!\n' +
    "We're bringing fully composable components â€” arrange them in any order.",
  ),
  new Section()
    .setComponents(new TextDisplay().setContent('A brief overview of components:'))
    .setAccessory(
      new Button().setStyle(ButtonStyle.Link).setLabel('Overview')
        .setURL('https://discord.com/developers/docs/components/overview'),
    ),
  new Separator(),
  new TextDisplay().setContent('-# Composed entirely from v2 components.'),
);

await ctx.write({ components: [container], files: [banner], flags: MessageFlags.IsComponentsV2 });
```

## Common patterns / gotchas

- **Forgetting the flag** is the #1 mistake. Every send AND every edit/update of a v2 message needs `flags: MessageFlags.IsComponentsV2`. Drop it on an edit and Discord rejects the request.
- **No top-level text/embeds/poll/stickers** alongside v2 components â€” Discord rejects the message. Move all text into `TextDisplay` (markdown supported, including `-#` small text and `##` headings).
- **Throwing `toJSON()`**: a `Section` with no accessory throws `SeyfertError('MISSING_ACCESSORY')`; a `MediaGalleryItem` with no media throws `SeyfertError('MISSING_MEDIA')`. Serialization happens inside `ctx.write`/`update`, so a half-built tree fails there â€” set the accessory/media before sending. Catch with `SeyfertError.is(err, 'MISSING_ACCESSORY')`.
- **Buttons in a Container**: interactive/link buttons go in an `ActionRow` child, or as a single `Section` accessory. `Container.addComponents` rejects a bare `Button` at the type level (allowed children: `ActionRow | TextDisplay | Section | MediaGallery | Separator | File`).
- **Spacing has only `Small`/`Large`.** Do not write `Spacing.Medium`/`Spacing.None` â€” they don't exist. `setDivider` defaults to `false` in the builder (despite the API comment saying "Defaults to true"), so pass it explicitly when you want a line.
- **Media references**: `attachment://<name>` resolves against a matching `AttachmentBuilder` passed in `files: [...]`; a plain URL is also accepted by `File`/`Thumbnail`/`MediaGalleryItem`. Names must match exactly.
- **`MediaGalleryItem` is a plain class** â€” no `setId`, no `BaseComponentBuilder`. Only container-level builders expose `setId` (component IDs are optional integers for targeting in edits).
- **All symbols import from the root `'seyfert'`** â€” no deep `'seyfert/lib/...'` imports needed.

## Doc vs Source Corrections

- Upstream MDX example 1 calls `ctx.write({ components: [container] })` WITHOUT a flag -> src requires `flags: MessageFlags.IsComponentsV2`; v2 components render only with the flag, and Seyfert never injects it (grep of `src/` shows `IsComponentsV2` defined only in channel.ts and never auto-applied). The MDX "File" tab and final "Example" DO include the flag (`1 << 15` / `MessageFlags.IsComponentsV2`).
- `Separator.setSpacing` JSDoc lists `'None' | 'Small' | 'Medium' | 'Large'`, but the actual `Spacing` enum has only `Small = 1` and `Large = 2`. Use only `Spacing.Small` / `Spacing.Large`.
- `Separator.setDivider` builder default is `false` (the API field comment claims "Defaults to true"); pass the value explicitly.
- Both `Section.setComponents(...)` and `addComponents(...)` exist and accept only `TextDisplay` children; the accessory (`Button`/`Thumbnail`) goes through `setAccessory`.
- Section-without-accessory and MediaGalleryItem-without-media THROW on `toJSON()` â€” the docs do not mention this.
- `Container.addComponents` does NOT accept a raw `Button` (or a `TextDisplay`-as-accessory). Wrap a button in an `ActionRow` or use it as a `Section` accessory.

## Agent Guidance

- Always set `flags: MessageFlags.IsComponentsV2` on every send AND every edit/update of a v2 message; this is the single most common omission.
- Build trees from `Container` for grouped layouts; use `Section` to pair `TextDisplay` text with one accessory (`Button` or `Thumbnail`). For a row of interactive buttons, wrap them in an `ActionRow`.
- Put all visible text in `TextDisplay` â€” a v2 message cannot also carry top-level `content`/`embeds`.
- Reference uploaded files with `attachment://<name>` in `File`/`Thumbnail`/`MediaGalleryItem` `.setMedia`, and pass the matching `AttachmentBuilder` in `files: [...]`.
- Guard against throwing `toJSON()`: every `Section` needs an accessory and every `MediaGalleryItem` needs media before serialization (i.e. before `ctx.write`).
- `Spacing` has only `Small`/`Large` â€” never `Spacing.Medium`/`Spacing.None`.
- `MediaGalleryItem` is a plain class (no `setId`); only container-level builders expose `setId`.
- All these symbols import from the root `'seyfert'`; no deep `'seyfert/lib/...'` imports are needed.
